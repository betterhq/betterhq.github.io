Aug 28, 2018

很好的运用了AOP和反射的范例，实现通用的统一缓存处理

AOP类:

这个类代理了带有 @ReadCache 注解的方法，并且实现了对这些方法的返回结果进行统一的Redis缓存。

这个类的关键在于 getCacheKey() 方法：

  • 这个方法里面先获取到切面代理的方法
  • 再获取方法所有参数的注解，获得二维数组
  • 再遍历二维数组，判断是否存在 @CacheParam 注解
  • 对 @CacheParam 注解标注的变量，先获取到 @CacheParam 注解中的 value 值，再获取到变量的 toString() 字符串，然后对RedisKey进行 format 格式化(怎么格式化细节看后面，是一个自定义注解很好的应用)

@CacheParam 注解类:

  • @ReadCache 注解类中的 cacheKey 参数表示缓存的RedisKey，timeout 参数表示过期时间
  • @CacheParam 注解类中 value 的含义是参数在整个RedisKey中的位置
